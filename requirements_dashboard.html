SECTION 1 — PLATFORM OVERVIEW & SCALE
India_GST is a multi-client, cloud-based GST compliance platform designed for chartered accountants, tax professionals, and finance teams managing 200+ GSTINs from a single workspace. Every architecture decision in this document is driven by the business requirements below.

Core Platform Capabilities

Capability Description Scale / Notes
Data Ingestion Upload and process large Excel datasets — Sales Register, Purchase Register, GSTR-2B downloads. Up to 10 lakh rows (1,000,000 rows) per GSTIN per period.
GST Returns Covered GSTR-1, GSTR-3B, GSTR-6 (ISD), GSTR-9, GSTR-9C, GSTR-7, IFF (QRMP). Comprehensive compliance coverage.
Reconciliation GSTR-2B vs ITC Register, GSTR-1 vs GSTR-3B vs Books, GSTR-9 & 9C mapping. Multi-way automated matching.
Report Generation Automated reconciliation reports — ITC matching, liability calculation, HSN analysis. Exportable to Excel/PDF.
PDF Extraction Extract data from scanned PDF invoices, GSTR-3B drafts via OCR. Smart document processing.
Analytics Dashboards Charts and trends per GSTIN and PAN-level consolidated view. Enterprise-wide visibility.
Multi-Client Portal 200+ GSTINs managed from a single platform. Scalable client management.
Role-Based Access L0 Super Admin → L1 Admin → L2 CA → L3 → L4 → C1 Client. Strict hierarchical data isolation.
Secure Document Store Excel files, PDFs, filed copies, working files — all versioned. Centralized compliance repository.
AI-Assisted GST Analysis ITC classification, anomaly detection, GSTR-9 mapping (Phase 2/3). Predictive risk management.
SECTION 2 — USER ROLES & RBAC ARCHITECTURE
Every user in India_GST has exactly one role. Roles are enforced at the API layer. Implementation Phase: 7 Days (Ends 02 May 2026).
Role Level Responsibilities & Access Scope Deadline
L0 Super Admin Developer Team senior partners at Accolet. Full system + dev control. Can do everything. 26 Apr 2026
L1 Admin Admin-level access without dev tools. Full org visibility across all CAs and clients. 28 Apr 2026
L2 CA Full access to own assigned clients. Review and approve returns. 30 Apr 2026
L3 Handles only CA-assigned clients. Prepare returns, upload invoices, save drafts. 01 May 2026
L4 Handles only explicitly assigned low-complexity clients. Prepare draft returns. 02 May 2026
C1 Client External client login. Views only their own GSTIN's returns, approves or raises queries. 03 May 2026
User Management Business Rules
  • Users are never permanently deleted — only deactivated. Hard delete is not permitted.
  • Every client user is linked to exactly one company GSTIN at creation time.
  • Auto-generated passwords: 12-character strong password shown once in popup; never stored in plaintext.
  • Only Super Admin can add, edit, deactivate users, and reset passwords.

SECTION 4 — MODULE 2: GSTR-3B MONTHLY SUMMARY RETURN
GSTR-3B captures outward liability, ITC claims, and net tax payable. Implementation Phase: 14 Days (Ends 30 May 2026).

Feature / Capability Priority Business Logic / Rationale Notes Deadline
Auto-populate from GSTR-1 data (Table 3.1 hard-lock) P1 Automatically pull liability figures from the filed GSTR-1 to prevent manual overrides, enforcing the new GSTN mandate for Table 3.1. Oct 2024 regulatory change 20 May 2026
IMS inward actions: Accept / Reject / Pending per invoice P1 Process vendor invoices via IMS. "Accept" flows to 2B; "Reject" blocks ITC; "Pending" defers ITC to the next month, optimising working capital. IMS compliance downloadable in Excel 20 May 2026
AI-driven 2B vs Purchase Register reconciliation P1 Use multi-pass fuzzy matching (exact, tolerance-based, pattern-based) to identify valid ITC claims even when vendor invoice numbers have typos. Multi-pass matching algorithm 20 May 2026

GSTR-2B fetch & ITC auto-fill into Table 4 P1 Fetch static 2B data on the 14th. Extract eligible ITC and auto-populate Table 4A(5), restricting ITC claims to strictly what is available in 2B. Core ITC automation 20 May 2026
RCM liability calculation — Reverse Charge Mechanism P1 Identify specific inward supplies (GTA, Legal) and auto-calculate RCM liability. Ensure it is simultaneously claimed as ITC in the same month. Mandatory field (RCM Template) 25 May 2026

Reconciliation Engine — Technical Requirements
  • Fetch Purchase Register (PR) from ERP or Excel upload.
  • Fetch CURRENT GSTR-2B from GSTN via direct API pull (GSP integration).
  • Multi-pass matching algorithm: Exact match → Tolerance match (configurable %) → Pattern-based invoice number match.
  • Result buckets: Matched / Mismatched (value difference) (Date Difference) (Pan Matching GSTIN Mismatching) / Missing in 2B / Missing in PR / Duplicate.
  • Auto-fill Table 4 of GSTR-3B from reconciled eligible ITC.
